# 単体テストマトリクス

> 様式: [unit-test-from-design/unit-test-matrix-template.md](../../../.claude/skills/unit-test-from-design/unit-test-matrix-template.md)
> モード: **draft**（Issue #3 実装と並行作成。実コードとの突合は finalize で実施）

各テストケースには **テストケース ID（TC-XXX 形式・3 桁ゼロ埋め）** を採番し、対応する受け入れ条件（AC-XXX）・業務ルール（BR-XXX）を必ず紐付ける。AC-XXX が無い基盤 Issue は「実装内容」項目を `IMPL-XX` として観点化する。

## 凡例

| 略号 | 正式名称 | 補足 |
| --- | --- | --- |
| TC-XXX | テストケース ID | 単体テストの一意識別子（3 桁ゼロ埋め） |
| AC-XXX | 受け入れ条件 | 要件定義 `functional/*.md` で定義 |
| BR-XXX | 業務ルール | 要件定義 `業務ルール.md` で定義 |
| IMPL-XX | 実装内容項目 | AC が無い基盤 Issue（#3）で `共通部品設計.md` の実装内容を観点化した一時 ID（Issue #3 内ローカル） |

## マトリクス

| TC-008 | IMPL-01 | StateConflictException（DomainException） | 異常系 | Unit | 生成時に httpStatus=409・code=CONFLICT を保持 | 例外が対応する httpStatus/code を返す |
| TC-009 | IMPL-01 | BidLimitExceededException（DomainException） | 境界値 | Unit | 生成時に httpStatus=409・code=BID_LIMIT_EXCEEDED を保持 | 例外が対応する httpStatus/code を返す |
| TC-010 | IMPL-01 | BidDeadlinePassedException（DomainException） | 境界値 | Unit | 生成時に httpStatus=409・code=BID_DEADLINE_PASSED を保持 | 例外が対応する httpStatus/code を返す |
| TC-011 | IMPL-01 | SetBidTenantMismatchException（DomainException） | 異常系 | Unit | 生成時に httpStatus=400・code=SET_BID_TENANT_MISMATCH を保持 | 例外が対応する httpStatus/code を返す |
| TC-012 | IMPL-01 | ResetTokenExpiredException（DomainException） | 境界値 | Unit | 生成時に httpStatus=400・code=RESET_TOKEN_EXPIRED を保持（期限ちょうど/直後） | 例外が対応する httpStatus/code を返す |
| TC-013 | IMPL-01 | RateLimitedException（DomainException） | 境界値 | Unit | 生成時に httpStatus=429・code=RATE_LIMITED を保持（上限ちょうど/超過） | 例外が対応する httpStatus/code を返す |
| TC-014 | IMPL-02 | GlobalExceptionHandler.handleDomainException | 正常系 | Unit | 任意の DomainException サブクラスを catch | ErrorResponse{code,message} を組み立て、例外の httpStatus で ResponseEntity を返す |
| TC-015 | IMPL-02 | GlobalExceptionHandler.handleDomainException | 異常系 | Unit | ValidationException（details 保持あり）を catch | ErrorResponse.details に field/reason が設定される |

| TC-018 | IMPL-03 | GlobalExceptionHandler.handleValidationException | 境界値 | Unit | 複数フィールドが同時に違反 | details 配列に違反フィールド数分の要素が含まれる |
| TC-019 | IMPL-04 | GlobalExceptionHandler.handleUnexpectedException | 異常系 | Unit | 未捕捉の RuntimeException が発生 | 500・code=INTERNAL_ERROR、スタックトレース等の内部情報を含まないレスポンス |
| TC-020 | IMPL-04 | GlobalExceptionHandler.handleUnexpectedException | 異常系 | Unit | 未捕捉例外発生時のログ出力 | ログにはスタックトレースが記録される（レスポンスには含めない） |
| TC-021 | IMPL-05 | SystemClock.now() | 正常系 | Unit | 現在時刻取得 | `LocalDateTime.now()` 相当の値を返す |
| TC-022 | IMPL-05 | Clock（interface） | 正常系 | Unit | テスト用固定 Clock 実装に差し替え | 呼び出し元が固定時刻で動作し、`Clock` 経由以外で時刻取得しない |
| TC-023 | IMPL-06 | TenantId/UserId/JobId（値オブジェクト） | 異常系 | Unit | null／空文字で生成 | IllegalArgumentException 等で不正値を拒否する |
| TC-024 | IMPL-06 | TenantId/UserId/JobId（値オブジェクト） | 境界値 | Unit | 同一値を保持する 2 インスタンスを比較 | equals/hashCode が値として一致する |
| TC-025 | IMPL-06 | TenantId/UserId/JobId（値オブジェクト） | 権限境界 | Unit | 異なる ID 種別（例: TenantId と UserId）の値を取り違えて使用しようとする経路 | 型が異なるため代入・比較ができず誤用を防止する |
| TC-026 | IMPL-07 | EnumMapper | 正常系 | Unit | ドメイン enum（例: JobStatus）→ OpenAPI 生成 inline enum へ変換 | 対応する値へ 1:1 変換される |
| TC-027 | IMPL-07 | EnumMapper | 異常系 | Unit | マッピング未定義の値を変換しようとする | 例外またはフォールバックで安全に処理される（無変換で通さない） |
| TC-028 | IMPL-07 | EnumMapper | 境界値 | Unit | null を入力 | null 安全に処理される（NPE を発生させない） |
| TC-029 | IMPL-08 | ロギング（業務例外） | 正常系 | Unit | 業務例外（400/404/409 系）発生時 | WARN レベルでログ出力される |
| TC-030 | IMPL-08 | ロギング（未捕捉例外） | 異常系 | Unit | 未捕捉例外発生時 | ERROR レベルでログ出力される |
| TC-031 | IMPL-08 | ロギング（機微情報マスク） | 正常系 | Unit | パスワード・トークン・氏名・電話番号・メールアドレスを含むリクエスト/レスポンスの処理 | 機微フィールドがログへ出力されない（マスクされる） |
| TC-032 | IMPL-08 | ロギング（MDC リクエスト ID） | 正常系 | Unit | 例外発生時のログ出力 | ログにリクエスト ID（MDC）が付与される |
| TC-033 | IMPL-09 | PageMeta 算出ロジック | 正常系 | Unit | 一覧系 operationId の結果を PageMeta（page/size/totalElements/totalPages）に変換 | 各フィールドが仕様どおりに算出される |
| TC-034 | IMPL-09 | PageMeta 算出ロジック | 境界値 | Unit | 該当件数 0 件・ちょうど 1 ページ分の件数 | totalPages が 0 または 1 として境界どおりに算出される |
| TC-035 | IMPL-10 | NotificationPublisher（Port） | 正常系 | Unit | 通知発火を Port 経由で呼び出す | テナント ID を宛先として通知が発行される（Port 実装はモック） |
| TC-036 | IMPL-11 | MailSender（Port、雛形） | 正常系 | Unit | Port インターフェースのメソッドシグネチャに準拠したスタブ実装を呼び出す | シグネチャどおりに呼び出し可能（実装本体は auth Issue 側で検証） |

「区分」は **正常系 / 異常系 / 境界値 / 権限境界** のいずれかを必ず指定し、各観点で合理的に該当する区分を網羅する。

- `check-test-matrix.sh`（ハードゲート）は draft モードでは実行しない（finalize で必須）。
