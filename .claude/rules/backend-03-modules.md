# バックエンド実装ルール 03 — モジュール構成と責務

## ルートパッケージ

```
com.example.logisticsmatching
```

## V1 モジュール一覧

```
src/main/java/com/example/logisticsmatching/
├─ LogisticsMatchingApplication.java
├─ auth/
├─ tenant/
├─ job/
├─ bid/
├─ deal/
├─ delivery/
├─ notification/
├─ query/
└─ shared/
```

---

## 各モジュールの責務

### auth — 認証

対象: ログイン・パスワードリセット・簡易ロック

責務:
- 認証ユースケース
- ロック状態・パスワード再設定トークン有効性の最小ドメイン
- 認証 API の入口

domain 化する: ログイン失敗回数・ロック状態・トークン有効性

---

### tenant — テナント / ユーザー管理

対象: 企業アカウント登録・テナント内ユーザー追加

責務:
- テナントとユーザー所属制約
- 同一企業重複登録判定の入口
- テナント単位の認可前提データの提供

domain 化する: テナント境界・ユーザー所属制約

---

### job — 案件登録管理

対象: 案件登録・編集・削除・ステータス管理

責務:
- 案件の状態遷移
- 編集 / 削除可能条件
- 論理削除状態の管理
- 成約後更新禁止（不変条件）

domain 化する: 案件ステータス・編集/削除可能条件・論理削除・成約後更新禁止

---

### bid — 応募 / セット応募

対象: 単体応募・応募編集・応募上限判定・締切判定・セット応募

責務:
- 応募制約（1 社 1 提案・上限 20 社・締切判定）
- セット応募制約（同一テナント・オール・オア・ナッシング）
- 初回応募時の案件ステータス変化
- 他社成約時の連動不成立判定

V1 で内包する: set-bid を bid 配下に置く（将来独立化の余地を残す）

domain 化する: 上記すべての制約ルール

---

### deal — 連絡 / 交渉 / 合意 / 成約

対象: メッセージ送信・金額交渉・最終条件提示・合意・成約スナップショット生成・他応募クローズ

責務:
- 成約に至る前後のユースケース
- 合意可能状態の判定
- 成約時の不可逆処理（スナップショット・他応募クローズ）

V1 で内包する: negotiation（連絡・交渉）を deal 配下に置く

domain 化する: 合意可能状態・成約成立条件・スナップショット生成・他応募クローズ条件・セット応募との連動

---

### delivery — 運送ステータス / 評価

対象: 運送開始・完了報告・完了確認・評価登録

責務:
- 成約後のステータス遷移（成約済 → 運送中 → 完了）
- 報告主体の制約
- 評価 1 回制約・双方評価完了時の状態変化

V1 で内包する: rating（評価）を delivery 配下に置く（将来独立化の余地を残す）

domain 化する: 遷移制約・実行主体制約・評価 1 回制約・状態変化

---

### notification — 通知

対象: 通知生成・通知一覧取得・既読化

責務:
- 通知の業務イベント接続
- 通知配信と永続化
- 既読処理

domain 化する: 発火イベントの定義・通知対象ユーザー/テナント範囲・通知種別の区分

---

### query — 参照系クエリ

対象: ダッシュボード・募集中一覧検索・取引履歴一覧・通知一覧取得

責務:
- 画面向け read model の取得
- 一覧系 API の整形
- 参照系の軽量実装

構成: `presentation / application / infrastructure` の 3 層。domain は不要。

制約:
- 書込み系の domain model に深入りしない
- CQRS の厳密な module 分離は V1 では不要
- 表示制約（テナント境界・競合情報非表示など）だけ最低限維持する

---

### shared — 共通部品

対象: 共通例外・共通識別子（`TenantId`, `UserId`, `JobId` など）・`Clock` interface・共通設定

ルール:
- 業務ロジックを shared に置かない
- shared に入れるのは本当に横断的なものだけに限定する

禁止:
- 業務ルール・業務判断を shared に追加する
- 特定モジュールの都合でその domain model を shared に移動する
